04
Why Bonito

Bonito is the control plane Everly needs

Everly has to be multi-model, multi-tenant, governed, auditable, and integration-ready. That is platform work. Bonito removes the undifferentiated rebuild.

Without Bonito
Every app wires providers, retries, rate limits, token logs, model fallback, and prompt governance itself.
Tenant knowledge, audit, cost, approvals, and memory become custom one-off code.
Adding channels and tools multiplies failure paths and increases security risk.
Vendor lock-in Weak audit No shared primitives
With Bonito
Gateway: one API for OpenAI, Anthropic, AWS, Azure, Google, and Groq routing.
Agents: prompts, model tiers, memory, KB, approvals, tools, and schedules per role.
Governance: cost, logs, audit, fallback, policy, and provider credentials stay centralized.
Tenant memory Model routing Approvals Audit trail
Architecture decision: production Everly should use one Bonito organization per client, plus a separate Everly Internal organization for the overseer. Each client gets isolated KBs, credentials, logs, API keys, and billing; overseer access is audited, read-only, service-role based, and aggregate or de-identified by default.

Build Scope

Prototype repo shipped now

The first repo proves the core experience and keeps the architecture honest: tenant data isolation, prompt split, Bonito gateway call, deterministic local fallback, and a workspace surface.

01

Workspace shell

A Boutique Sage interface with tenant overview, weekly brief, tenant corpus, activity logs, and a live Everly conversation panel.

02

Bonito chat endpoint

Next.js API route builds the tenant-scoped system prompt, retrieves tenant facts first, then calls Bonito's OpenAI-compatible gateway when configured.

03

Deployable agent config

bonito.yaml defines team, customer, fact extractor, and overseer agents with model tiers, prompts, tenant memory, and approval posture.

04

Honest v1 boundary

This is not a full CRM, phone stack, or OAuth suite. It is the right base for wiring real integrations without losing the product principles.
