# Token-Based Pricing Shift — Bonito Positioning

## Summary

The industry is shifting from flat-rate subscriptions (GitHub Copilot at $10-19/seat/mo) to **token-based, usage-metered pricing**. GitHub Copilot now has premium request limits and overage charges. OpenAI, Anthropic, Google, AWS — all charge per token with different rates per model. This shift makes AI spend **variable and unpredictable**, which is exactly the problem Bonito solves.

---

## Video Talking Points

### 1. "The subscription safety net is gone"
- Flat pricing meant predictable budgets. Token-based means your AI bill is now like your cloud bill — variable, spiky, and hard to forecast.
- A single runaway agent or poorly-written prompt loop can burn through thousands in tokens overnight.
- Bonito gives you spend caps, real-time cost tracking, and alerts before you blow budget.

### 2. "Every provider prices differently — good luck comparing"
- OpenAI charges X per million tokens, Anthropic charges Y, Bedrock charges Z (and varies by region).
- Same question, wildly different cost depending on where you route it.
- Bonito normalizes this — one dashboard, one cost view, across all 6 providers. Apples-to-apples.

### 3. "Routing is now a money lever, not just a performance lever"
- When tokens have a price, choosing the right model for each request = direct savings.
- A simple summarization doesn't need GPT-4o or Opus — route it to Haiku or Groq and save 90%.
- Bonito's routing policies (cost-optimized, latency-optimized, balanced) do this automatically.

### 4. "Governance went from nice-to-have to mandatory"
- Flat seats had built-in spending limits. Token-based has none unless you build them.
- Enterprises need: model allow-lists, per-team budgets, rate limiting, audit trails.
- This is table stakes in Bonito — it's already built.

### 5. "This validates the control plane"
- The punchline: **the more usage-based AI pricing becomes, the more you need infrastructure to manage that usage.**
- It's the same story as cloud compute — AWS bills got unpredictable, so CloudHealth/Spot/Vantage emerged.
- Bonito is that layer for AI. One control plane to connect, govern, route, and optimize across every provider.

### 6. "The multi-provider future is locked in"
- Token-based pricing forces comparison shopping. No one's locking into one provider when prices differ 10x between models.
- Enterprises will use 3-5 providers minimum. They need a single abstraction layer.
- That's Bonito's entire thesis — and the market is moving toward us, not away.
