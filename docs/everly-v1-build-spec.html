01 / SUMMARY

What this brief argues.

The Everly V1 PRD specifies 36 requirements across tenant lifecycle, identity, onboarding, knowledge base, conversational interface, capabilities, permissions, voice and brand discipline, pricing, observability, and training-data substrate. The two-week timeline is aggressive but feasible — conditional on not rebuilding the AI plumbing from scratch.

Everly V1 is structurally identical to a wheel-model agent architecture we have already built in production: a per-tenant orchestrator agent, a shared roster of specialist capabilities, a per-tenant knowledge base, session memory, and a voice layer — all running on a multi-provider gateway with observability, governance, and audit baked in.

That substrate is Bonito. Everly runs on it free. The cost is hosting the Everly application layer on top — the web app, the database, the Stripe integration, the marketing surface. That bill is small.

This brief covers:

  • The architecture, mapped from the PRD onto Bonito's agent framework
  • The hosting and infra costs, line by line, with Bonito as a zero-cost dependency
  • The capability-to-agent topology (which specialist owns which V1 capability)
  • The Lindy.ai competitive picture — what they actually do, where they fall short of Everly's positioning, and what we can credibly deliver that they cannot
  • The build sequence for a two-week V1, with the load-bearing principles preserved

The bottom layer — gateway, agents, KB, memory, voice — is the part that takes months to build well. It's already built. What Bonito eats — and what that's worth

Gateway, agents, KB, memory, voice, observability. Standalone, those subsystems would cost $400-$2,000+/mo at modest scale (LLM provider fees, vector DB hosting, voice API subscriptions, observability tooling) and 12-16 weeks of engineering to build to production quality. At V1 scale that's roughly $5K-$24K saved annually on infra alone, plus 3-4 months of build time you don't spend.

At 100 tenants paying $750-1,000/mo, you're running gross-margin-positive on day one. The cost stack stays predictable as you scale because the AI layer is fixed-cost from Everly's perspective.

05 / BUILD SEQUENCE

The two-week V1 path.

Working backwards from "first paying customer in under an hour from signup." The load-bearing principles (multi-tenant isolation, voice discipline, ask-before-acting, no-card onboarding, transparent data) are preserved. UI polish and conversational onboarding can slip to V1.1 if the timeline forces a cut.

1

Foundation (days 1-3)

Next.js scaffold, Postgres schema (tenants, users, configs), NextAuth magic-link auth, public sign-up page (no card, Req 4), tenant provisioning service (Req 1), per-tenant Bonito org + orchestrator created at signup via the same pattern OuchGPT uses today.

2

KB pipeline (days 3-5)

Document upload UI, file ingestion service → Bonito KB API (Req 8, 11), citation rendering in answers (Req 12), per-tenant isolation enforced at the Bonito org boundary, admin transparency view stub (Req 10).

3

Conversational surface + voice (days 5-8)

Chat UI streaming responses (Req 15), per-user thread persistence (Req 16), voice mode toggle (Req 14) wired to Bonito's ElevenLabs/Whisper gateway, voice persona configured to brand spec, three-layer voice enforcement (system prompt + runtime sanitizer + build-time linter, Req 27).

4

Specialists + capabilities (days 6-11)

Five specialist agents deployed via bonito.yaml — marketing-drafter, scheduler, call-summarizer, data-hygienist, email-drafter. Orchestrator wired to delegate via invoke_agent. Ask-before-acting enforced through Bonito's approval-queue primitive (Req 23). "Worth noting" sentinel parser in the web UI (Req 17). No-go list enforcement in system prompt + tool absence (Req 25).

5

Integrations (days 8-12)

Google Workspace OAuth (Calendar read + Gmail draft push), HubSpot OAuth (read + propose-only writes). Tools added to specialist allowlists. Per-tenant OAuth tokens stored in Bonito secrets vault.

6

Conversational onboarding (days 10-13)

15-20 min discovery interview, structured question script (CPO deliverable), output feeds the per-tenant KB. Available in text and voice. Per the PRD, this is non-negotiable for V1.

7

Admin + observability + billing (days 11-13)

Admin surface (Req 33): tenant config, team invites with content tiers (Req 7), integration management, KB document management, transparency view. Per-tenant cost/tokens/latency dashboard fed by Bonito gateway metrics (Req 34). Engagement signal for quiet-failure detection (Req 35). Stripe card capture flow for day-30 conversion (Req 31), usage-tier metering (Req 32). Tenant deletion + export (Req 3).

8

Voice eval + ship (day 14)

15-prompt voice eval suite (Req 29) running in CI, blocking ship on failure. Vocal eval for voice mode brand alignment before ship. AI disclosure verified on every outbound surface (Req 28).

Honest read on the timeline

Two weeks is achievable for a wedge V1 that ships exactly what the PRD names as non-negotiable. Anything cosmetic or convenience-oriented — visual analytics polish, integration polish beyond what's listed, multi-modal extras — slides to V1.1. The load-bearing pieces (multi-tenant isolation, ask-before-acting, voice discipline, transparent data, no-card onboarding) stay in V1 because they are the wedge.

If we discover during build that we need an extra week, the call is between CPO and engineering. The wedge is not negotiable; the polish is.

06 / LINDY.AI

What they actually do — and where they don't reach.

Lindy is the most-cited individual-professional AI assistant. They have real product surface and real revenue. They are also positioned in a different category than Everly is going after. The honest read.

What Lindy actually offers

  • Workflow-builder model. Users build "lindies" (AI workflows) using a canvas-style builder. Each lindy is a multi-step automation: trigger, steps, actions.
  • Pre-built templates. 1,000+ templated workflows for individual professionals: meeting prep, lead qualification, content drafting, calendar coordination.
  • Integration breadth. 50+ integrations (Gmail, Calendar, Slack, HubSpot, Salesforce, Notion, Linear, others).
  • Voice agents. Lindy Phone offers outbound and inbound call handling — they sit in the voice-agent category in addition to the workflow category.
  • Pricing. Free tier, then Pro (~$49/mo), Business (~$299/mo), Enterprise custom. Credit card required to start beyond the free tier.
  • Funding posture. $50M+ raised. Real revenue, growing. Not a competitor we can dismiss.

Where Lindy's claims don't hold up — relative to Everly's wedge

The honest framing

Lindy is broader. Everly is deeper. Lindy wins on integration count and outbound voice today. Everly wins on every dimension that defines the wedge: the SMB business identity, zero-config onboarding, trust-before-access, voice discipline, ask-before-acting as a runtime constraint, and the growth arc framing.

The competitive question isn't "can we match Lindy's surface area." It's "can the SMB market we're targeting be acquired by the wedge before Lindy notices the lane." The PRD's two-week timeline assumes the answer is yes.

07 / RISK COVERAGE

The PRD's risk modes, mapped to Bonito primitives.

Each of the named V1 risks has an existing platform-level mitigation in Bonito. Nothing here is a feature we build for Everly — it's already shipped infrastructure that Everly inherits.

Voice failure

Sanitizer + linter + eval gate

Three-layer voice enforcement runs on every reply. The build-time linter blocks em-dash glyphs from being shipped in any tracked source file. The runtime sanitizer strips voice-violation patterns before reply emits. The voice eval suite (15 prompts) gates CI. Voice failures cannot reach production.

Permission breach

Approval queue at the platform layer

Bonito's approval queue primitive (already shipped) inspects every tool call for risk classification. External-effect tools cannot fire without an approval. Risk assessment, auto-approve conditions, timeout handling are platform features, not Everly application code. The agent loop cannot violate ask-before-acting even on a model error.

Data trust breach

Per-tenant isolation by default

Every Bonito org is fully isolated. KB content, agent configs, request logs, audit trail are all per-tenant boundaries. Cross-tenant queries return nothing. The transparency view reads from the same source as the agent — what's shown is what exists. There is no second source of truth to misalign.

Competitive response

Compounding substrate from day one

Bonito's training-data substrate (Req 36) captures every interaction inside a tenant boundary from the first signup. Per-tenant first, cross-tenant aggregation post-V1. The longer Everly runs, the larger the moat. Larger competitors who arrive later cannot rebuild this substrate without rebuilding the product.

Quiet failure

Engagement signal in the dashboard

Per-tenant engagement metric defined as "meaningful interactions" per period. Drops below threshold during trial → Slack alert + dashboard alert to the Everly team. The PRD's named higher-probability failure mode is detected within five days, not at the churn-log point.

Cost predictability

Real-time per-tenant cost capture

Every model call writes provider, model, tokens, latency, cost — attributed to a tenant ID. Unit economics per tier are observable from day one. The pricing tier spec (Req 32) can be tuned with real data instead of assumptions.

Two questions to decide together.

The build path is laid out. Before kicking off, two open items the founders and engineering decide jointly.

Question 1

Voice persona finalization

The CPO drives brand characterization; engineering recommends TTS candidates. Decision needed before Day 8 of the build. Bonito ships with ElevenLabs voices wired; a 30-minute joint call to pick the voice unblocks Step 3 of the sequence.

Question 2

Pricing tier spec

PRD Req 32 leaves tier definitions open (starter, growth, top end at $750-$1,000). Bonito's per-tenant cost data will inform tier triggers, but the customer-facing tier feature gates are a CPO call. Decision needed before Day 11.
